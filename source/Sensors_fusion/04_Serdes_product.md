# **车载计算平台摄像头带宽测算与 SerDes 接口选型**

---

## **一、核心结论（摘要）**

| 维度            | 结论                                                    |
| ------------- | ----------------------------------------------------- |
| **系统规模**      | 全车 **12 路摄像头**（前 3 / 周 4 / 环 4 / 后 1），满足 L2+/L3 级智能驾驶 |
| **总带宽**       | 全系统 **未压缩数据总带宽 ≈ 23.4 Gbps**                          |
| **带宽瓶颈**      | 前视 8MP RAW 帧带宽最高（3.73 Gbps），决定 SerDes 规格              |
| **SerDes 选型** | **GMSL2（6 Gbps）最优**：满足带宽、成本与生态要求                      |
| **SoC 推荐**    | 高通 SA8295P（8 路）、地平线征程 5（6 路）                          |
| **工程原则**      | **性能可达、成本可控、接口预留、安全独占**                               |

---

## **二、系统架构：输入—处理—输出三层模型**

### **1. 输入层：摄像头感知矩阵**

共 12 路摄像头：

* 前视 3 路（高分辨率 RAW）
* 周视 4 路
* 环视 4 路
* 后视 1 路

覆盖高速 ADAS、城市 NOA、泊车等场景。

### **2. 处理层：解串器 + SoC**

* **解串器（Deserializer）**：负责 SerDes→CSI-2、时钟恢复、聚合、I2C 透传等
* **SoC（域控制器）**：负责 ISP/NPU/GPU/CPU 协同处理

### **3. 输出层**

输出到仪表、中控、HUD 等，实现人机交互和增强现实辅助驾驶。

---

## **三、摄像头配置：按功能匹配分辨率与格式**

| 类别           | 数量 | 分辨率       | 帧率    | 位深    | 格式     | 用途          |
| ------------ | -- | --------- | ----- | ----- | ------ | ----------- |
| 前视（长焦/广角/鱼眼） | 3  | 3840×2160 | 30fps | 12bit | RAW    | 车道、车辆识别、AEB |
| 周视           | 4  | 1920×1080 | 30fps | 16bit | YUV422 | 盲区监控        |
| 环视           | 4  | 1920×1300 | 30fps | 16bit | YUV422 | 泊车、全景拼接     |
| 后视           | 1  | 1920×1080 | 30fps | 16bit | YUV422 | 倒车辅助        |

**原则：**

* 前视保留 RAW → 保留动态范围，利于 AEB / ACC
* 周/环/后视采用 YUV → 节省带宽、图像链路成本更低

---

## **四、带宽计算：统一计算模型**

### **计算公式**

> **带宽 (Gbps) = 分辨率 × 位深 × 帧率 × 1.25（协议与同步开销）**

1.25 开销因子：包含 MIPI/SerDes 同步、控制信号占比，是工程计算刚需。

### **计算结果汇总**

| 摄像头类型  | 单路带宽           | 合计              |
| ------ | -------------- | --------------- |
| 前视（3）  | **3.732 Gbps** | 11.196 Gbps     |
| 周视（4）  | 1.244 Gbps     | 4.976 Gbps      |
| 环视（4）  | 1.500 Gbps     | 6.000 Gbps      |
| 后视（1）  | 1.244 Gbps     | 1.244 Gbps      |
| **总计** | —              | **23.416 Gbps** |

**设计启示**：

* 前视数据量巨大，是整个系统的**瓶颈项**
* 单路峰值必须小于单路 SerDes 的极限带宽

## **五、SerDes 接口选型：为何 GMSL2 是最优解**

| 接口类型         | 单路带宽       | 成熟度      | 成本 | 结论                 |
| ------------ | ---------- | -------- | -- | ------------------ |
| GMSL1        | 3.12 Gbps  | 高        | 中  | ❌ 不满足前视 3.73 Gbps  |
| **GMSL2**    | **6 Gbps** | **非常成熟** | 中高 | **✔ 行业主流、性价比最高**   |
| GMSL3        | 12 Gbps    | 新        | 高  | ✔ 面向 4K/60fps 未来应用 |
| FPD-Link III | 2.97 Gbps  | 高        | 中  | ❌ 带宽不足             |
| FPD-Link IV  | 12 Gbps    | 成熟中      | 高  | ✔ 高端车型             |

**关键论证**：

* 前视单路 3.73 Gbps > GMSL1 上限 → **必须采用 GMSL2 或更高版本**
* GMSL2（6 Gbps）足够 → **成本 & 生态最优**

## **六、域控制器（SoC）选型建议**

| 芯片             | GMSL2 支持数 | 总带宽     | 推荐理由            |
| -------------- | --------- | ------- | --------------- |
| **高通 SA8295P** | 8         | 48 Gbps | **生态强，8 路足够扩展** |
| NVIDIA Orin X  | 12        | 72 Gbps | 针对 L3+/Robotaxi |
| **地平线征程 5**    | 6         | 36 Gbps | **性价比高，满足当前需求** |
| 黑芝麻 A1000      | 8         | 48 Gbps | 国产优选            |

### **接口分配推荐（基于 GMSL2）**

* 前视 3 路 → **3×独占接口（安全冗余）**
* 周视 4 路 → **2×接口（2路/接口）**
* 环视 4 + 后视 1 → **1×接口（总带宽 5.72 Gbps）**

**总需求：6 路**
→ SA8295P/征程5 均可满足。

## **七、解串器技术要点**

解串器的关键作用不只是“串并转换”，还包括：

* 多路聚合（减少 SoC CSI 通道数量）
* 时钟恢复（保证帧同步）
* I2C 透传（远程升级/标定的基础）
* 错误检测（温度、误码率）

主流器件：

* ADI/Maxim：MAX96712（GMSL2）、MAX9286
* TI：DS90UB954/964（FPD-Link）

| 趋势方向   | 说明                                    |

## **九、选型**

| 任务     | 建议                       |
| ------ | ------------------------ |
| 产品定义   | 明确 L2+/L3，采用 12 路摄像头标准组合 |
| 接口选型   | **必须用 GMSL2 或更高版本**      |
| SoC 选择 | 推荐 SA8295P 或征程 5         |
| 成本控制   | 让接口利用率保持在 **70–80%**     |
| 安全冗余   | 前视三路必须独占链路               |
| 未来扩展   | 预留 2–3 个接口用于 OTA 或新增摄像头  |

**一句话总结**

> **“采用 GMSL2 实现 12 路高清摄像头、总带宽约 23.4Gbps，选 SA8295P/征程5 方案并预留 30% 接口余量，是当前 L2+/L3 智能驾驶最稳健的方案。”**
